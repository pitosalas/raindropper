# F18 — Tag Lint: Propose Singular/Plural and Similar-Stem Merges Interactively

## T1 — Add plural/singular detection helper
**Status**: done

Add `_plural_singular_proposals(tags)` to `raindropper/actions.py`.

- `tags` is a list of tag objects with `_id` and `count`.
- Build a set of all tag names.
- For each tag, test whether stripping a plural suffix yields another existing tag:
  - strip trailing `s` (but not if result is < 2 chars)
  - strip trailing `es`
  - strip trailing `ies` and append `y`
- If a match is found and source != target, emit a proposal dict:
  `{"kind": "plural", "source": source_id, "target": target_id, "source_count": N, "target_count": M}`
- Avoid duplicate pairs (track seen pairs as frozensets).
- Return list of proposals.

---

## T2 — Add similar-stem detection helper
**Status**: done

Add `_similar_stem_proposals(tags, existing_proposals)` to `raindropper/actions.py`.

- `tags` is the full list of tag objects.
- `existing_proposals` is the list from T1 (to skip pairs already proposed).
- For each unique pair of tags, compute `difflib.SequenceMatcher(None, a, b).ratio()`.
- If ratio ≥ 0.85 and the pair is not already in existing_proposals, emit:
  `{"kind": "similar", "source": lower_count_id, "target": higher_count_id, "source_count": N, "target_count": M}`
  (if counts equal, alphabetically later becomes source)
- Avoid O(n²) blowup: only compare pairs where both tags have len ≥ 3.
- Return list of proposals.

## T3 — Add `merge_tag` helper on client
**Status**: done

Add `merge_tag(self, source_tag, target_tag)` to `RaindropClient` in `raindropper/client.py`.

- Fetch all bookmarks that have `source_tag` via `fetch_bookmarks_by_tag`.
- For each bookmark, if `target_tag` not already in its tags: replace tags list with source removed and target added, call `update_bookmark_tags`.
- Delete `source_tag` via `delete_tags([source_tag])`.
- Return count of bookmarks updated.

## T4 — Add `lint_tags` action to `actions.py`
**Status**: done

Add `lint_tags(client, selection_set)` to `raindropper/actions.py`.

- Fetch all tags.
- Generate proposals = plural proposals + similar-stem proposals (T1 + T2).
- If no proposals, print "No tag lint proposals found." and return.
- Process proposals in batches of 10:
  - Print numbered list: `  N. [plural] MERGE 'source' → 'target'  (source: X uses, target: Y uses)`
  - Prompt: `Accept (numbers, 'a'=all, Enter=skip, 'q'=quit): `
  - Parse input: `'a'` → accept all in batch; numbers → accept those indices; Enter → skip; `'q'` → stop.
  - For each accepted proposal, call `client.merge_tag(source, target)`, accumulate total count.
  - After each batch print: `Merged N tag(s) so far.`
- Print final: `Done. Merged N tag(s) total.`
